# 🧠 AI 构建认知体系 · 随身判断卡（1 页版）

> **用途**：当你看到任何 AI 新概念 / 新框架 / 新口号时，
> 用这 1 页，在 **30 秒内判断：要不要学、怎么用、放哪一层**。

---

## 一、三条不可动摇的公理（底座）

### 公理 1
**LLM 是概率生成器，不是认知主体**  
→ 不理解、不负责、不自知

### 公理 2
**一切“智能感”来自流程，而不是模型**  
→ 状态、顺序、规则、校验

### 公理 3
**不确定性必须被围起来**  
→ 用规则 / 验证 / 兜底

❗ 任何理念若违背以上任意一条 → 直接忽略

---

## 二、四层结构（认知操作系统）

```
L1 理念层（Why）   —— 本质判断，不会过时
L2 原语层（What）  —— 能力最小单元
L3 架构层（How）   —— 工程组织方式
L4 应用层（Where） —— 真实业务价值
```

⚠️ 不同层的东西 **不能混着讨论**

---

## 三、理念层 · 五条铁律（长期有效）

1. **模型不可信，流程才可信**
2. **自动化 ≠ 自主**
3. **越关键的动作，越不能交给模型**
4. **Agent 是工程对象，不是生命体**
5. **可控性 > 智能感**

---

## 四、原语层 · 只承认这 6 个能力

```
1. Parse   —— 解析 / 结构化输入
2. Plan    —— 拆解步骤 / 策略
3. Act     —— 执行动作（工具 / 函数）
4. Memory  —— 状态 / 上下文
5. Check   —— 校验 / 评估
6. Loop    —— 重试 / 分支 / 终止
```

### 快速映射
- **Skills ≈ Act（函数化、确定性）**
- **Agent ≈ Loop + Memory + Plan（流程控制）**
- **LLM 只该出现在 Parse / Plan / Check**

👉 没有“新原语”，只有“新包装”

---

## 五、架构层 · 推荐长期稳定结构

```
Orchestrator（Agent / 状态机）
  ├─ 规则 & 边界
  ├─ 流程控制
  └─ 调用 Skills（Functions）
           ↓
      外部系统 / API
```

### 边界原则
- Agent **不做能力**
- Skills **不做决策**
- LLM **不直接干活**

---

## 六、应用层 · 四问反推法（必问）

1. 用户最终要什么结果？
2. 哪一步最不确定？ → LLM
3. 哪一步最危险？ → Rules
4. 哪一步必须可复现？ → Code

---

## 七、理念口号快速翻译器

| 口号 | 工程翻译 |
|---|---|
| Skills 代替 Agent | 削弱自主 Loop，强化确定性函数 |
| 多 Agent 协作 | 多流程节点 + 明确输入输出 |
| 自主智能体 | 自动化状态机 |
| 推理增强 | 更多规则 + 校验 |

---

## 八、30 秒判断过滤器（终极版）

看到任何新概念，只问 5 个问题：

1. 它动的是 **理念层** 还是 **架构层**？
2. 它引入了新的 **原语** 吗？（几乎不会）
3. 它是在 **降低** 还是 **放大** 不确定性？
4. 控制权是上移（人/规则）还是下移（模型）？
5. 不用它，我能不能用现有原语实现？

≥ 2 个 ❌ → 不学
