# 文章配图说明

> 本文件夹存放文章配图及配图Prompt

**创建日期**：2025-10-26

---

## 📁 文件夹结构

```
images/
├── README.md                        # 本说明文档
└── 文章主题关键词/                  # 每篇文章一个文件夹
    ├── image-1.png                  # 配图文件
    ├── image-2.png
    ├── ...
    └── prompts.md                   # 配图Prompt文档
```

---

## 🎨 配图规范

### 文件夹命名
- 格式：`YYYY-MM-DD-文章主题关键词/`
- 示例：`2025-10-26-Claude-Code-评测/`

### 图片文件命名
- 格式：`image-序号.png` 或 `image-序号.jpg`
- 示例：`image-1.png`, `image-2.png`
- 或使用描述性名称：`cover.png`, `comparison.png`

### 推荐尺寸
- **横版**：1200x675 (16:9) - 适合横屏展示
- **方形**：1080x1080 (1:1) - 适合封面
- **竖版**：1080x1350 (4:5) - 适合手机竖屏

---

## 📝 配图Prompt文档格式

每个文章文件夹下必须包含 `prompts.md`，记录所有配图的生成Prompt。

**模板格式**：
```markdown
# 文章配图Prompt - [文章标题]

## 配图1：封面图

### 图片描述（中文）
[描述这张图的内容、风格、用途]

### Prompt Version 1 (通用版)
```
[英文Prompt]
// 中文注释：适用于大多数AI图片生成工具
```

### Prompt Version 2 (细节增强版)
```
[英文Prompt，更详细]
// 中文注释：适用于需要更多细节的场景
```

### Prompt Version 3 (风格化版)
```
[英文Prompt，强调艺术风格]
// 中文注释：适用于需要特定视觉风格的场景
```

### 建议尺寸
- 方形：1080x1080 (1:1)

### 备选方案
[如果AI生成效果不好，建议从哪里找替代图片]

### 实际使用
- [ ] 已生成
- [ ] 已插入文章
- 文件名：`image-1.png`
```

4. **截图/官方素材**

   - 自己的截图
   - 需标注来源

## ✅ 配图流程

### 1. 确定配图需求
- 文章需要几张图
- 每张图的位置和作用
- 推荐：5-8张

### 2. 创建文件夹
```bash
mkdir images/YYYY-MM-DD-文章主题/
```

### 3. 生成Prompt文档
- AI会自动生成 `prompts.md`
- 包含每张图的3个版本Prompt

### 4. 生成/获取图片
- 使用Prompt生成图片
- 或从其他来源获取
- 保存到对应文件夹

### 5. 插入文章
- 在Markdown中插入图片
- 使用相对路径或绝对路径

---

## 📌 Prompt设计原则

### 好的Prompt特征
- ✅ 描述清晰具体
- ✅ 包含风格指导
- ✅ 适合文章主题
- ✅ 避免版权问题

### 三个版本的区别
- **Version 1**：简洁通用，快速生成
- **Version 2**：细节丰富，质量更高
- **Version 3**：风格化强，艺术性强

---

## 💡 使用建议

### 配图数量
- 建议：5-8张
- 最少：3张（封面+2张内容图）
- 视文章长度调整

### 配图位置
- ✅ 文章开头（封面/引入）
- ✅ 每个大章节开头
- ✅ 重要概念说明
- ✅ 对比展示
- ❌ 避免过度配图

### 风格一致性
- 同一篇文章的配图风格要统一
- 颜色、风格、视觉元素保持一致

## ⚠️ 注意事项

### 版权问题
- AI生成的图片：通常可商用（查看具体工具条款）
- 公共领域作品：可自由使用
- 免费图库：查看具体授权
- 他人作品：需获得授权

### 文件大小
- 单张图片建议 < 1MB
- 保持清晰度的同时控制大小
- 必要时压缩图片

### 文件格式
- 推荐：PNG（透明背景）或 JPG（照片）
- 避免：BMP、TIFF等大体积格式
