## 频道定位与视觉策略

**频道核心定位：技术拆解师 - 让人人都懂技术**

| 原则 | 说明 | 视觉体现 |
|------|------|----------|
| **拆解感** | 把复杂变简单，层层剥开 | 分步卡片、拆解动画、积木隐喻 |
| **亲和力** | 技术不高冷，人人可理解 | 卡通人物、友好配色、生活化类比 |
| **专业度** | 内容硬核，讲解到位 | 信息图表、结构化布局、清晰标注 |
| **记忆点** | 一眼认出是你的内容 | 固定角色、统一风格、品牌色 |

**品牌视觉识别**：
- **主角色**：戴眼镜的技术男生（视觉代言人）
- **核心隐喻**：积木（可拆解）、流程图（有步骤）、放大镜（深入看）
- **品牌感**：专业但不高冷，深入但不晦涩

## 风格A：拆解者风格（Tech Breakdown）

**核心理念**：聚焦技术本质，纯粹的拆解可视化

**适用场景**：文章封面、概念解释、技术原理拆解、对比分析

| 元素 | 规范 |
|------|------|
| **背景** | 深色科技感（深蓝/深紫/渐变），可带科技纹理 |
| **主色调** | 深色底 + 高亮标题（黄/青/白） |
| **核心元素** | 拆解可视化（积木、层级、分解图）+ 3D图标元素 |
| **视觉焦点** | 3D图标、发光效果、模块化结构（**无人物**） |
| **信息密度** | 中等，突出1-2个核心概念 |

**拆解感视觉元素**（无人物版）：
- 3D立体积木/模块（可组合可拆分，LEGO风格）
- 层级展开图（洋葱剥皮式，分层展示）
- 发光的放大镜图标（深入细节，聚焦核心）
- 流程箭头 + 标注（步骤清晰）
- X光透视效果 + 高亮线条（看穿本质）
- 齿轮/机械元素（系统运作）
- 爆炸视图/拆解动效（结构展示）

**背景变体**：
- 深蓝 + 电路板纹理：硬件/底层技术
- 深紫 + 粒子效果：AI/未来技术
- 深灰 + 网格线条：架构/系统设计
- 渐变 + 光效：综合/入门内容

**视觉焦点替代方案**（替代人物角色）：
- 发光的放大镜图标（探索/深入）
- 3D齿轮组合（运作/机制）
- 聚光灯效果（突出重点）
- 方向箭头 + 标注（指引/说明）
- 模块化拼图（组合/拆解）

## 风格B：图解说明风（Visual Guide）

**核心理念**：一图胜千言，让复杂流程一目了然

**适用场景**：流程说明、步骤教程、对比分析、知识总结

| 元素 | 规范 |
|------|------|
| **背景** | 浅色/白色，干净简洁 |
| **主色调** | 白底 + 黑色边框卡片 |
| **核心元素** | 卡片+箭头+图标 |
| **人物风格** | 手绘感卡通，温暖亲和 |
| **信息密度** | 高，多步骤多元素 |

**图解核心元素**（保持不变）：
- 编号卡片（1→2→3步骤感）
- 黑色箭头（清晰流向）
- 时间标注（效率对比）
- 图标+标签（图文结合）
- 对比分隔（Before/After）

**配色方案（语义化）**：
- 🔵 蓝色：AI/技术/工具
- 🟠 橙色：人/行动/重点
- 🟢 绿色：成果/好处/完成
- ⚫ 黑色：边框/箭头/结构
- ⚪ 白色：背景/留白

## 风格选择指南

**核心原则**：风格A吸引点击，风格B辅助理解

| 场景类型 | 推荐风格 | 使用模板 |
|----------|----------|----------|
| 封面/缩略图 | 风格A | A5. 视频封面场景 |
| 痛点/问题 | 风格A | A3. 问题共鸣场景 |
| 概念类比 | 风格A | A1. 概念拆解场景 |
| 原理透视 | 风格A | A2. 原理透视场景 |
| 对比拆解 | 风格A | A4. 对比拆解场景 |
| 流程步骤 | 风格B | B1. 步骤拆解卡片 |
| 效率对比 | 风格B | B2. 效率对比卡片 |
| 知识结构 | 风格B | B3. 知识结构图 |
| 工具推荐 | 风格B | B4. 工具推荐卡片 |
| 总结回顾 | 风格B | B5. 总结回顾图 |

**单个视频的配图组合**：
1. **封面**：风格A（吸引点击）
2. **问题引入**：风格A（引发共鸣）
3. **概念解释**：风格A（聚焦核心）
4. **步骤拆解**：风格B（清晰跟随）
5. **总结回顾**：风格B（一图看懂）

## 提示词优化原则

| 原则 | 说明 |
|------|------|
| **减少中文文字** | AI生成中文常出错，简化或省略，后期用PS/Figma添加 |
| **用图标代替文字** | 能用图标表达的不用文字 |
| **保留简单数字** | 概率、百分比等数字可以保留 |
| **一段式描述** | 豆包提示词用自然语言一段式描述，不分块 |
| **风格词前置** | 在描述末尾加入明确的风格词 |

## 配图数量建议

| 视频时长 | 建议配图数 | 分布 |
|----------|-----------|------|
| 5分钟以下 | 3-5张 | 封面+2-3张内容图 |
| 5-10分钟 | 6-10张 | 封面+每章节1-2张 |
| 10-15分钟 | 10-15张 | 封面+每2分钟1张 |

**最小可行配图（MVP）**：
- 1张封面（必须）
- 1张核心概念图（必须）
- 1张总结图（推荐）
